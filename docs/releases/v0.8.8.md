# Release v0.8.8

## 🐛 Bug Fixes

### Fixed Teller Certificate Loading Priority

Changed certificate loading to prefer base64-encoded environment variables over file paths, which is the recommended production approach.

**What Was Fixed:**
- **Base64-first approach** - Now tries base64 decoding first (for env vars), then falls back to file paths
- **Better path detection** - Improved logic to detect file paths (checks for `/`, `./`, `.secure/` prefixes)
- **Clearer error handling** - Better error messages when neither base64 nor file path works

**Why This Matters:**
Production deployments typically use base64-encoded certificates in environment variables, not file paths. The previous approach tried file paths first, which could fail unnecessarily. Now it correctly prioritizes the production-friendly approach.

**Technical Details:**
- Reversed the order: try base64 decode first, then file paths
- Base64 certificates are typically 1000+ characters, file paths are <500
- Improved file path detection with multiple prefix checks
- Better error messages distinguish between decode failures and file read failures
