# Release v0.8.18

## 🐛 Bug Fixes

### Fixed Static File Serving in Docker Containers

Fixed a critical issue where CSS and other static files were returning 404 errors in production Docker deployments.

**What Was Fixed:**
- **Smart path detection** - Server now checks for both `/static` (Docker container path) and `static` (local development path)
- **Absolute path support** - Uses `/static` when available in containerized environments
- **Backward compatible** - Still works in local development with relative `static` directory

**Why This Matters:**
The Dockerfile copies static files to `/static` in the container, but the server was looking for `static` relative to the working directory. This caused all CSS files to fail loading, breaking the UI in production.

**Technical Details:**
- Added `os.Stat("/static")` check to detect containerized environment
- Falls back to relative `static` path for local development
- Single file change in `cmd/server/main.go` with minimal impact
