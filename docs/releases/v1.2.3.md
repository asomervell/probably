## 🐛 Bug Fixes

### Resolve Critical itoa Bug and Improve Code Quality

Fixed a critical bug where custom itoa() function was producing reversed strings, and improved overall code quality with better context handling and timezone standardization.

**What's New:**
- Fixed itoa() producing reversed strings by using strconv.Itoa
- Fixed context.Background() usage in transfer_matcher, now properly propagates ctx
- Standardized timezone handling to UTC in aggregator for consistent date calculations
- Simplified and documented SQL logic in pulse calculator for spendable accounts

**Why This Matters:**
The itoa bug was causing incorrect string conversions throughout the application, potentially leading to data corruption or incorrect calculations. Proper context propagation ensures that request cancellation and timeouts work correctly. UTC timezone standardization prevents timezone-related bugs in date calculations.

**Technical Details:**
- Fixed `internal/models/transaction.go` to use strconv.Itoa instead of custom itoa()
- Updated `internal/sync/transfer_matcher.go` and `go/internal/sync/transfer_matcher.go` for proper context propagation
- Standardized timezone handling in `internal/insights/aggregator.go`
- Enhanced `internal/pulse/calculator.go` with better SQL documentation and logic
