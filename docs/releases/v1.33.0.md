# Release v1.33.0

**Date:** 2026-01-03
**Previous Version:** v1.32.0

This release introduces significant performance improvements to the transactions page through HTMX lazy loading, making the initial page load faster and providing a smoother user experience.

## ✨ Features

### HTMX Lazy Loading for Transactions Page

The transactions page has been redesigned with a split architecture that dramatically improves initial load times:

- **Shell + Content Architecture**: The page now loads in two phases:
  - **Shell** (immediate): Page layout, header, search bar, and filter tabs placeholder render instantly
  - **Content** (lazy-loaded): Transaction list, pagination, and filter tabs populate via HTMX after the shell loads

- **Loading Skeleton Animations**: While content loads, users see elegant animated skeleton placeholders that match the final layout, providing visual feedback that content is loading

- **New Content Endpoint**: Added `/transactions/content` route that handles all transaction list rendering, filter tab counts, and pagination logic separately from the page shell

- **Improved User Experience**: Users can see the page structure immediately and start interacting with search/filters while transaction data loads in the background

## 🧹 Code Cleanup

- **Removed Unused MCP CDN Debug Logging**: Cleaned up unused debug logging code related to MCP CDN operations

## Technical Details

- The `TransactionsList` handler now renders a lightweight shell with HTMX attributes (`hx-get`, `hx-trigger="load"`) that automatically fetches content from `/transactions/content`
- Filter tabs use HTMX Out-of-Band (OOB) swapping to update the placeholder after content loads
- Loading skeletons are rendered inline in the shell, providing immediate visual feedback
- All existing functionality (filtering, search, pagination, bulk actions) continues to work seamlessly with the new architecture
