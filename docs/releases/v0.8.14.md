# Release v0.8.14

## ✨ Features

### GitHub Releases for All Deployments

This release enables GitHub releases for every deployment, not just tagged releases, providing better visibility into what's deployed.

**What's New:**
- **Releases for all deployments** - Every push to main now creates a GitHub release
- **Version tag support** - Workflow now triggers on version tags (v*) in addition to main branch pushes
- **Automated release notes** - Uses GitHub's automated release notes generation
- **Build artifacts** - Server binary is attached to every release

**Why This Matters:**
Previously, only manually tagged releases created GitHub releases. Now every deployment gets a release, making it easy to see what version is currently deployed and download the exact binary that's running in production.

**Technical Details:**
- Added `tags: - 'v*'` trigger to workflow
- Removed conditional `if: steps.version.outputs.is_release == 'true'` from release step
- Added `generate_release_notes: true` for automatic changelog generation
- Server binary is attached to all releases for easy download
