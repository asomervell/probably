# v1.28.0

**Date:** January 2, 2026  
**Previous Version:** v1.27.0  
**Commit Range:** v1.27.0..v1.28.0

## Summary

This release focuses on improving the bank connection experience, particularly for Plaid users, with better status visibility, connection management, and visual polish. We've added connection status monitoring, improved account management workflows, and enhanced the overall user experience when managing connected banks.

## ✨ Features

### Connection Status Monitoring for Plaid Accounts
- Added real-time connection status tracking for Plaid bank connections
- Connection status banners now appear on the connected accounts page and account detail pages when action is required
- Status types include:
  - **Login Required** - Connection needs re-authentication (highest priority, shown in red)
  - **Pending Disconnect** - Connection will be disconnected soon (shown in red)
  - **Pending Expiration** - Connection will expire soon (shown in yellow)
  - **New Accounts Available** - Additional accounts can be added to the connection (shown in blue)
- Banners include direct action buttons to reconnect or add new accounts
- Status is automatically updated during sync operations

### Delete All Accounts for Institution
- Added "Delete All Accounts" button for each connected institution (Teller, Plaid, Akahu)
- Allows bulk deletion of all accounts for a specific bank/institution
- Includes confirmation dialog showing the number of accounts and transactions that will be deleted
- Useful for cleaning up duplicate connections or removing entire bank integrations
- Works across all providers (Teller, Plaid, Akahu) and matches accounts by institution name

### Improved Account Selection for Plaid
- Enabled account selection mode when reconnecting Plaid connections
- Users can now choose which accounts to add when new accounts become available
- Streamlined workflow for adding additional accounts to existing connections

### Plaid Institution Logo Support
- Added automatic fetching and storage of Plaid institution logos
- Logos are downloaded from Plaid's base64 data format and stored in cloud storage
- Institution logos now display on the connected accounts page for Plaid connections
- Consistent with existing Teller logo support

### Enhanced Redirects and Navigation
- All sync and disconnect operations now redirect to `/settings/banks` instead of `/accounts`
- Users stay on the connected accounts management screen after making changes
- Improved workflow continuity when managing bank connections
- Better alignment with the settings-focused bank management experience

## 🐛 Bug Fixes

### Plaid Duplicate Connection Detection
- Fixed issue where users could accidentally connect the same Plaid Item multiple times
- System now detects duplicate connections and shows a helpful error message
- Offers option to reconnect existing connection instead of creating duplicates
- Prevents data duplication and confusion

### Transaction List Tag Pills Styling
- Fixed transaction list tag pills to use consistent default styling
- Pills now use: `bg-primary/10 text-primary border border-primary/20 rounded-full`
- Improved visual consistency across the application
- Better readability and visual hierarchy

### Connection Status Handling
- Improved error handling for Plaid connection status updates
- Better detection of connection issues during sync operations
- More reliable status updates and banner display

## Technical Details

### Database Migration
- **Migration 056**: Added `connection_status` column to `accounts` table
  - Stores status values: `update_required`, `pending_expiration`, `pending_disconnect`, `new_accounts_available`
  - Added index on `(ledger_id, connection_status)` for efficient queries
  - Supports connection status tracking across all providers

### API Changes
- Added `SettingsDeleteInstitutionAccounts` handler for bulk account deletion
- Enhanced Plaid sync service to fetch and store institution logos
- Added `StoreBase64Logo` method to `LogoStore` for handling Plaid's base64 logo format
- Updated `GetInstitution` to request optional metadata including logos

### Code Improvements
- Standardized redirect behavior across all sync/disconnect operations
- Improved Plaid provider integration with connection status tracking
- Enhanced error messages and user feedback for connection issues
- Better handling of Plaid Item errors and reconnection flows

### Dependencies
- Updated `esbuild` from 0.21.5 to 0.27.2
- Updated `vite` from 5.4.21 to 7.3.0
- Improved build tooling and development experience

### Documentation
- Updated Teller troubleshooting documentation with improved error handling information
- Better guidance for resolving connection issues
- Added Plaid integration documentation

## User Impact

This release significantly improves the bank connection management experience:

- **Better Visibility**: Users can now see at a glance when their bank connections need attention
- **Easier Management**: Bulk deletion makes it simple to clean up duplicate or unwanted connections
- **Clearer Actions**: Status banners provide direct action buttons to resolve connection issues
- **Visual Polish**: Institution logos and improved styling make the interface more professional
- **Smoother Workflow**: Improved redirects keep users in the right context when managing connections

Users with Plaid connections will particularly benefit from the new status monitoring and improved error handling, making it easier to maintain healthy bank connections and resolve issues quickly.
