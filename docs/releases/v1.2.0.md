## ✨ Features

### Migrate from Merchants to Entities and Update Related Functionality

Refactored the codebase to use a more flexible "entities" concept instead of "merchants", enabling better support for various types of financial entities including merchants, bills, subscriptions, and more.

**What's New:**
- Renamed merchant-related code to use entity terminology
- Updated database schema to support entities
- Enhanced entity search and matching functionality
- Updated enrichment pipeline to work with entities
- Improved entity logo extraction and storage
- Updated handlers and API endpoints for entity management

**Why This Matters:**
The migration from merchants to entities provides a more flexible foundation for the application. Entities can represent merchants, bills, subscriptions, and other financial relationships, making the system more extensible and better aligned with how users think about their finances.

**Technical Details:**
- Updated `internal/db/migrations/036_entities.sql` and `038_detected_recurring_entity_id.sql`
- Refactored `internal/handlers/merchants.go` to `internal/handlers/entities.go`
- Updated `internal/enrichment/firecrawl_client.go` and `internal/enrichment/logo_client.go`
- Enhanced `internal/insights/aggregator.go` and `internal/insights/analyzer.go` for entity support
- Updated `internal/handlers/patterns.go` and `internal/handlers/transactions.go` for entity relationships
- Modified `internal/categorize/prompts.go` and `internal/categorize/groq.go` for entity context
