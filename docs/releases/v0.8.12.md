# Release v0.8.12

## ✨ Features

### Major CI/CD Refactor: Separate Build and Deploy Jobs

This release refactors the deployment workflow into separate build and deploy jobs with improved caching, security, and efficiency.

**What's New:**
- **Separated build and deploy** - Build and deployment are now independent jobs that can run in parallel
- **Go module caching** - Added caching for Go build cache and module downloads to speed up builds
- **Distroless Docker images** - Switched to distroless base image for smaller, more secure containers
- **Artifact management** - Binary is built once and uploaded as artifact, then downloaded in deploy job
- **Semantic versioning support** - Added version calculation that detects tagged releases vs. development builds

**Why This Matters:**
Separating build and deploy allows for better caching, faster builds, and more reliable deployments. The distroless base image reduces attack surface and image size. Artifact management ensures the exact same binary that was built is deployed.

**Technical Details:**
- Split single job into `build` and `deploy` jobs with `needs: build` dependency
- Added Go module cache with `actions/cache@v4`
- Switched from full Debian base to `gcr.io/distroless/static-debian12:nonroot`
- Binary built in build job, uploaded as artifact, downloaded in deploy job
- Version detection: tagged releases use tag name, others use `v0.0.0-YYYYMMDD-SHA` format
- Full git history fetched (`fetch-depth: 0`) for proper version calculation
