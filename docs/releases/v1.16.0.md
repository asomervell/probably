# Release v1.16.0

**Date:** 2026-01-01  
**Previous Version:** v1.15.0

## Summary

This release introduces a major navigation restructure and the new "My Life" feature for managing personal relationships. The navigation has been reorganized for better discoverability, and several legacy pages have been removed.

## Features

### My Life Settings Page

New settings page for defining the important people, work relationships, and assets in your life:

- **People**: Track partners, family, and friends for better transfer identification
- **Work**: Define employers, clients, and contractors for income categorization
- **Assets**: Manage vehicles, properties, and pets for expense tracking
- Relationships improve AI categorization accuracy

See [My Life](../features/my-life.md) for full documentation.

### Navigation Restructure

Improved navigation grouping for better organization:

**Main Menu:**
- AI & Insights: Pulse, Intelligence, Chat
- Data & Transactions: Statements, Patterns, Transactions, Transfers, Accounts
- Settings

**Settings Menu:**
- Account: Profile, My Life
- Data: Tags, Connected Banks, Ledger
- System: Billing, Data Backup, Danger Zone

**Notable Changes:**
- Transfers moved to main menu (no longer in settings)
- Settings reorganized with logical groupings
- Mobile navigation updated to match

### Danger Zone Rebuild Operations

New maintenance operations in Settings → Danger Zone:

- **Full Reprocessing**: Re-enrich and re-categorize all transactions
- **Pattern Reprocessing**: Re-detect all recurring patterns
- **Recalculate Balances**: Recalculate opening balances for all accounts

## Removed Features

The following pages have been removed:

- `/rules` - AI categorization rules page
- `/settings/ai` - AI settings page
- `/settings/p2p` - Peer-to-peer settings page

## Technical Changes

### Database
- New migration `049_relationships.sql` for storing relationships

### Models
- New `Relationship` model with full CRUD operations

### UI
- New `IconHeart` for My Life menu item
- Updated navigation in `base.go` with divider groupings
