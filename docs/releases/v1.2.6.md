## 🐛 Bug Fixes

### Prevent Retrying Vertex AI Permission Errors to Reduce Costs

Added error handling to prevent retrying Vertex AI permission errors, which was causing excessive API costs from failed authentication attempts.

**What's New:**
- Added NonRetryableError type for permission/auth errors
- Detect 403/PERMISSION_DENIED errors in Vertex AI calls
- Stop worker from retrying non-retryable errors
- Add explicit service account configuration to Cloud Run deployment
- Add cost mitigation documentation and diagnostic scripts

**Why This Matters:**
When Vertex AI returns permission errors (403), retrying these requests doesn't help and only increases costs. This fix prevents the worker from wasting API calls on errors that won't resolve through retries, significantly reducing unnecessary costs. The diagnostic scripts help identify and fix permission issues quickly.

**Technical Details:**
- Added `NonRetryableError` type in `internal/processing/router.go`
- Enhanced error detection in Vertex AI calls to identify permission errors
- Updated `internal/processing/worker.go` to skip retries for non-retryable errors
- Added service account configuration to `.github/workflows/deploy.yml`
- Created diagnostic scripts: `scripts/check-billing-and-final-fix.sh` and `scripts/verify-runtime-auth.sh`
- Added documentation in `COST_MITIGATION.md` and `URGENT_STOP_COSTS.md`
