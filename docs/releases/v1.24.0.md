# v1.24.0

**Date:** January 27, 2025  
**Previous Version:** v1.23.0  
**Commit Range:** v1.23.0..v1.24.0

## Summary

This release introduces two major features: **ChatGPT App integration** enabling users to access their financial data through ChatGPT, and a comprehensive UI redesign with the new "brick-basil" theme. The ChatGPT App integration uses OAuth 2.1 with PKCE for secure authentication and provides rich, interactive financial widgets. The UI redesign includes a complete component system migration and significant improvements to UI consistency and accessibility.

## ✨ Features

### ChatGPT App Integration

**MCP Server for ChatGPT Apps**
- Full integration with ChatGPT using Model Context Protocol (MCP)
- OAuth 2.1 with PKCE authentication for secure access
- Dynamic client registration - ChatGPT automatically registers itself
- Integrated into main server at `/mcp` endpoint (recommended) or standalone mode

**Financial Tools Available in ChatGPT**
- `get_spending_summary` - Spending breakdown by category and time period
- `get_account_balances` - Net worth, assets, and liabilities overview
- `ask_question` - AI-powered financial insights and analysis
- `get_spending_trends` - Time series spending trends
- `get_recurring_patterns` - Subscriptions and recurring bills detection
- `search_transactions` - Transaction search with natural language queries
- `get_financial_overview` - Comprehensive financial dashboard

**Interactive UI Widgets**
- React-based widgets using OpenAI Apps SDK UI
- Rich visualizations for financial data
- Responsive design with light/dark theme support
- Widgets include: Spending Summary, Account Balances, Financial Overview, Recurring Patterns, Spending Trends, Transaction Search, and AI-powered Q&A

**Security & Configuration**
- Environment variable-based configuration (no hardcoded URLs)
- OAuth token store with API key mapping for secure internal API access
- Audit logging for all MCP operations
- Configurable data retention policies
- Subscription checks enforced at tool execution level

**Developer Experience**
- Comprehensive setup documentation
- Standalone MCP server option for testing
- UI bundle build system with Vite and TypeScript
- Process polyfill for browser compatibility
- Local development support with `/mcp-ui/` route for serving bundles

### Brick-Basil Theme Implementation

**New Color Scheme**
- Light mode: Neutral warm tones with cool undertones matching Cursor aesthetic
  - Background: `#F6F6F3` (warm neutral)
  - Foreground: `#14120b` (deep warm brown)
- Dark mode: Complementary dark tones for better contrast and readability
- Color format migrated from oklch to hex for better browser compatibility
- Theme automatically adapts to system preferences (light/dark mode)

**Visual Design Updates**
- Removed card shadows for a flatter, cleaner appearance
- Improved color consistency across all components
- Better contrast ratios for accessibility
- Cursor-inspired aesthetic with modern, minimal design

  - `EntityCard` - Card component for displaying entities

**Component Consistency**
- All components now use consistent shadcn styling

### UI Consistency Improvements

**Responsive Design Fixes**
- Fixed narrow input widths on settings pages by adding `min-w-48` to select dropdowns
- Improved mobile layout consistency
- Better spacing and padding throughout the application

**Interactive Elements**
- Added `cursor-pointer` to all clickable elements (buttons, links, onclick handlers)
- Improved hover states and transitions
- Better visual feedback for user interactions

**Layout Refinements**
- Fixed excessive padding in Summary cards by using `CardContent` instead of `CardContentFull`
- Entity detail pages now use shadcn components consistently (Card, Input, Textarea, Badge, Button, TableRow, TableCell)
- Transactions table wrapped in Card component while keeping header card-free
- Transaction rows use shadcn TableRow and TableCell components for consistency

### Accessibility and Standards

**Unit System Migration**
- Replaced all pixel-based widths (`w-[Npx]`, `min-w-[Npx]`, `max-w-[Npx]`) with rem-based Tailwind utilities
- Replaced pixel-based heights (`min-h-[Npx]`, `max-h-[Npx]`) with rem-based utilities
- Replaced inline pixel width styles with Tailwind classes
- Used `w-px` and `h-px` for 1px separators instead of arbitrary values
- Better scalability and consistency across different screen sizes and zoom levels

**Color Standardization**
- Removed hardcoded colors (indigo, amber, red, emerald) in favor of theme variables
- All colors now use CSS custom properties for easy theme customization
- Improved color contrast for better readability

## Upgrade Notes

This release includes both UI improvements and new ChatGPT App integration. Database migrations will run automatically.

**For Users:**
- No action required for UI improvements
- ChatGPT App integration is optional - configure via environment variables if you want to use it

**For Developers:**
- All `components.*` imports have been replaced with `shadcn.*`
- Custom components are now in the `shadcn/` directory
- Theme colors are now defined in CSS custom properties
- Pixel-based sizing has been replaced with rem-based Tailwind classes
- **NEW**: MCP server requires `MCP_BASE_URL` and `MCP_UI_CDN_URL` (or `CDN_DOMAIN`) environment variables
- **NEW**: UI bundles must be built with `cd cmd/ui-bundles && npm run build` before deployment
