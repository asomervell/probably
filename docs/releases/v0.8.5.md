# Release v0.8.5

## ✨ Features

### Add Validation for DATABASE_URL Secret in Deployment Workflow

Added validation to ensure DATABASE_URL secret is configured before deployment.

**What's New:**
- Added validation check for DATABASE_URL secret in deployment workflow
- Deployment will fail early if DATABASE_URL is not configured
- Prevents deployment to Cloud Run without database configuration

**Why This Matters:**
Early validation prevents deploying a broken application that can't connect to the database. This saves time and makes deployment failures more obvious and easier to debug.

**Technical Details:**
- Added validation step in `.github/workflows/deploy.yml`
- Checks for DATABASE_URL secret before deployment
- Fails fast with clear error message if missing
