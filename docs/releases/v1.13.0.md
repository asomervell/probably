# v1.13.0 - Entity-First Pattern Detection

**Release Date:** January 1, 2026

## Overview

This release introduces a major improvement to pattern detection, replacing random batch processing with an entity-first approach that dramatically improves accuracy.

## What's New

### Smarter Pattern Detection

The system now analyzes **all transactions from each merchant together**, giving the AI complete context to understand your recurring payments.

**Before:** Random batches of transactions → Poor accuracy, missed patterns  
**After:** Full merchant history → 45+ patterns detected, multiple patterns per entity

### Multiple Patterns Per Entity

Entities can now have multiple distinct patterns. For example, Apple transactions might be detected as:
- Apple iCloud Storage (monthly subscription)
- Apple One (monthly subscription)  
- App Store purchases (variable)

### Pattern Detail Pages

Click any pattern to see:
- **AI Reasoning**: Why the system identified it as a pattern
- **Transaction History**: All transactions included
- **Statistics**: Confidence, occurrences, total spent
- **Next Expected**: When the next charge is expected

### Past Patterns Section

Patterns that haven't occurred recently are now shown in a separate "Past Patterns" section, keeping your current patterns organized.

## Bug Fixes

- Fixed credit card outflow detection (positive amounts for liability accounts)

## Technical Improvements

- Entity-based worker claiming with database-level locking
- Pattern hints stored on entities for future detection context
- New migration for pattern hints (048)

## Documentation

- New [Pattern Detection](../features/patterns.md) feature documentation

## Upgrade Notes

No action required. Pattern detection will automatically use the new entity-first approach when you click "Refresh Detection" on the Patterns page.
