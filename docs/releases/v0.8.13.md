# Release v0.8.13

## ✨ Features

### Enhanced Logo Enrichment and CSS Build Pipeline

This release significantly improves logo enrichment reliability and adds proper CSS building to the CI/CD pipeline.

**What's New:**
- **Logo.dev always attempted** - Logo.dev is now always tried when configured, even without a website (can search by merchant name)
- **CSS build in CI/CD** - Added Tailwind CSS build step to GitHub Actions workflow
- **Static file artifacts** - CSS and static files are now properly uploaded/downloaded as artifacts
- **Improved error handling** - Added explicit nil checks for logoClient and logoStore throughout enrichment worker
- **GitHub secrets management** - Added helper scripts for managing GitHub variables and secrets

**Why This Matters:**
Previously, logo.dev was only attempted when a website was available. Now it can search by merchant name, dramatically improving logo coverage. The CSS build fix ensures production deployments have properly minified CSS instead of missing styles.

**Technical Details:**
- Added Tailwind CSS binary download and build step in workflow
- Static files (including built CSS) are uploaded as artifacts and downloaded in deploy job
- Enhanced enrichment worker with nil-safe logo store access
- Logo.dev can now search by merchant name without requiring a domain
- Added scripts: `check-github-vars.sh`, `set-github-secrets.sh`, `set-missing-github-vars.sh`, `verify-github-vars.sh`
- Updated project documentation with semver commit message guidelines
