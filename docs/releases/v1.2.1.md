## 🐛 Bug Fixes

### Remove Unused Enrichment Code and Fix Vertex AI SDK Integration

Cleaned up legacy enrichment code and fixed critical issues with Vertex AI SDK integration that were preventing proper LLM calls.

**What's New:**
- Removed old enrichment worker (`internal/enrichment/worker.go`)
- Removed deprecated enrichment status tracking functions
- Fixed Vertex AI SDK system instruction role (use RoleUser instead of RoleModel)
- Added comprehensive error logging for Vertex AI calls
- Enhanced worker error logging with transaction counts and error details
- Updated utility commands to use categorization_status instead of enrichment_status

**Why This Matters:**
Removing unused code reduces maintenance burden and potential confusion. The Vertex AI SDK fix ensures that system instructions are properly sent, which is critical for getting accurate LLM responses. Better error logging helps diagnose issues more quickly when they occur.

**Technical Details:**
- Removed `internal/enrichment/worker.go` and related enrichment status functions
- Fixed `internal/processing/router.go` to use correct Vertex AI SDK role
- Updated `cmd/server/main.go` and `cmd/util/main.go` to remove enrichment references
- Enhanced error logging throughout processing pipeline
- Added documentation in `docs/PROCESSING_PIPELINE.md` and `docs/VERTEX_AI_SETUP.md`
- Updated dependencies in `go.mod` and `go.sum`
