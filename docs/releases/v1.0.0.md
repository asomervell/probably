## ✨ Features

### Remove Bud Transaction Enrichment Integration

Removed the Bud transaction enrichment integration as part of the v1.0.0 major release, simplifying the codebase and focusing on core functionality.

**What's New:**
- Removed Bud client and all related enrichment code
- Removed Bud category mapping functionality
- Cleaned up configuration options related to Bud
- Updated processing pipeline to remove Bud dependencies
- Added database migration to remove Bud-related columns

**Why This Matters:**
Removing the Bud integration simplifies the codebase and reduces maintenance overhead. This allows the team to focus on core features and alternative enrichment methods that better serve users' needs.

**Technical Details:**
- Removed `internal/enrichment/bud_client.go` and related files
- Deleted `internal/models/bud_category_mapping.go` and `internal/models/bud_customer.go`
- Added migration `037_remove_bud.sql` to clean up database schema
- Updated `internal/config/config.go` to remove Bud configuration
- Cleaned up `internal/processing/worker.go` and `internal/processing/llm.go`
- Updated `internal/handlers/merchants.go` and `internal/handlers/transactions.go`
