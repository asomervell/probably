# Release v1.14.0

**Date:** 2026-01-01  
**Previous Version:** v1.13.0

## Summary

This release modernizes the Pulse dashboard's upcoming bills calculation and adds frontend date localization. The system now computes recurring bills dynamically from transaction metadata, replacing the legacy `detected_recurring` table approach.

## Features

### Pulse Dashboard Migration to PatternStore

The Pulse dashboard's "Upcoming Bills" feature now uses the new pattern aggregation system:

- **Dynamic Pattern Calculation**: Upcoming bills computed in real-time from transaction metadata
- **Better Pattern Matching**: Uses `PatternStore.GetRecurringBills()` for flexible pattern detection
- **Confidence Filtering**: Only patterns with 50%+ confidence shown in upcoming bills

### Frontend Date Localization

Added automatic date localization throughout the application:

- **Browser Locale Support**: Dates formatted according to browser locale (e.g., "Jan 15" or "15 Jan")
- **Reusable Script**: Elements with `data-date="YYYY-MM-DD"` are automatically formatted
- **HTMX Compatible**: Dates auto-localize on dynamic content loads

## Deprecated Endpoints

The following legacy endpoints now return HTTP 410 (Gone):

- `PatternsAcknowledge` → Use `PatternsDismiss` instead
- `PatternsCancel` → Use `PatternsDismiss` instead  
- `PatternsPause` → Patterns are now computed dynamically

## Technical Changes

- Removed ~400 lines of unused `DetectedRecurring` render functions
- Pulse calculator uses `PatternStore` instead of `detected_recurring` table
- Pattern aggregation computed on-the-fly from transaction metadata

## Migration Notes

- The `detected_recurring` table still exists but is no longer used for Pulse
- Changes to transaction patterns are immediately reflected in the dashboard
- No user action required - migration is automatic
