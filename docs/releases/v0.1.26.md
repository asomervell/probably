# Release v0.1.26

## 🐛 Bug Fixes

### Fix: Use vars.GCP_PROJECT_ID Instead of Secrets

Fixed GitHub Actions workflow to use variables instead of secrets for GCP_PROJECT_ID.

**What's Fixed:**
- Changed from `secrets.GCP_PROJECT_ID` to `vars.GCP_PROJECT_ID` in workflow
- Uses GitHub repository variables instead of secrets for non-sensitive configuration

**Why This Matters:**
Repository variables are more appropriate for non-sensitive configuration values like project IDs. This simplifies secret management and follows best practices.

**Technical Details:**
- Updated `.github/workflows/deploy.yml` to use `vars.GCP_PROJECT_ID`
- Repository variables are easier to manage than secrets for public configuration
