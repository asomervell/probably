# v1.32.0 - JS to HTMX/CSS Migration

**Release Date:** January 2026

## Overview

This release represents a significant architectural improvement, migrating the frontend from JavaScript-heavy components to modern, native HTML/CSS solutions powered by HTMX. We've eliminated approximately 485 lines of custom JavaScript while maintaining all functionality and improving performance, accessibility, and maintainability.

## What's New

### Frontend Modernization

#### Toast Notifications
Server-rendered toasts with CSS-only auto-dismiss animations. Toasts automatically fade out after a configurable duration without requiring JavaScript.

#### Dialogs and Modals
Native HTML `<dialog>` element provides built-in escape key handling, focus trapping, and backdrop management with better accessibility.

#### Sheets and Drawers
CSS checkbox hack enables smooth slide-out panels (mobile settings sheet, history drawer) with CSS transitions.

#### Dropdown Menus
Native `<details>` element replaces custom JavaScript dropdowns with built-in keyboard navigation and screen reader support.

#### Navigation
Server-side active state detection ensures navigation links highlight correctly based on current route without client-side JavaScript.

#### Expandable Sections
Tag trees in Statements page use native `<details>` for collapsible sections with better semantic HTML structure.

#### Combobox Components
Toggle functionality handled by native `<details>` element while retaining client-side filtering for search.

### HTMX Upgrade
Updated from version 1.9.10 to 2.0.3 for better SSE support and improved performance.

## Technical Details

### What Remains as JavaScript
- **Voice Chat**: WebRTC audio capture, WebSocket streaming, audio playback
- **Chart.js**: Data visualization for financial charts  
- **WebAuthn**: Browser authentication API for passkeys
- **Plaid/Teller SDKs**: Third-party banking integration
- **Theme Detection**: FOUC prevention requires sync execution

### Architecture Improvements
- Reduced ~485 lines of custom JavaScript
- Native browser implementations improve performance
- Better accessibility through semantic HTML elements
- Improved testability with server-rendered components
