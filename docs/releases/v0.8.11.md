# Release v0.8.11

## 🐛 Bug Fixes

### Removed Local Storage Fallback - Cloud Storage Required

Removed the local storage fallback to ensure all deployments use cloud storage (GCS/S3) for reliability and consistency.

**What Was Fixed:**
- **Cloud storage required** - Logo client now requires cloud storage configuration, no local fallback
- **Removed local storage logic** - Eliminated development-only local storage path
- **Clearer error messages** - Better error when cloud storage is not configured

**Why This Matters:**
Local storage doesn't work in production environments like Cloud Run (read-only filesystem). Removing the fallback ensures all deployments are properly configured with cloud storage from the start, preventing runtime failures.

**Technical Details:**
- Removed local storage initialization from LogoClient
- Removed directory creation logic from LogoStore (cloud storage doesn't need it)
- Updated error messages to clearly indicate cloud storage is required
- Simplified storage initialization code
