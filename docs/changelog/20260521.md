**Verification that all three are already fixed in current code:**

   - `annotateToolResponse()` (`simple.go:202`) embeds function name into JSON payload
   - `inferToolResponseNameFromPayload()` (`api.go:1253`) recovers name from annotated payload
   - `sanitizeFunctionResponseNames()` (`api.go:1224`) last-resort fallback to `"search_entities"`
   - Called before every Vertex AI request (`api.go:440`, `api.go:840`)

2. **GEMINI_THOUGHT_SIGNATURE** — Fixed by explicit thinking suppression:
   - `isThinkingModel()` (`api.go:1296`) detects `gemini-3.x` and `gemini-2.5` models
   - When tools active OR tool history present: `ThinkingBudget=0` applied (`api.go:424`, `api.go:825`)
   - `thoughtSignaturesConsistent()` (`api.go:1300`) guards against preserving partial signatures
   - Mirrored in all three Vertex call paths (`callVertexAI`, `callVertexAIStreaming`, `callVertexAIChatToolsStreaming`)

3. **OTEL_SPAN_LIMIT** — Fixed in `otel.go:79-80`:
   - `MaxExportBatchSize(50)` — batches ≤50 spans (well under PostHog's 100-span limit)
   - `MaxQueueSize(75)` — caps in-flight queue; shutdown flush never exceeds one batch
