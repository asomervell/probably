### HAM-5045 — Fix stdlog bridge to catch all error-level messages

**File**: `internal/observability/stdlog_bridge.go`

The `" error"` keyword (leading space) missed:
- Messages starting with "error" or "Error:"
- Messages like "encoding/json error" (slash before space)

Also missing: `"failed"` keyword, very common in error-level log output.

**Change**: `" error"` → `"error"`, removed the now-redundant `"error:"` case, added `"failed"`.
