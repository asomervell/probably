### New finding: deploy workflow pipefail bug

Discovered `.github/actions/` directory exists (prior sessions incorrectly noted "no CI workflows"). Repo has `deploy.yml` + `semver` and `release-notes` composite actions.

**Root cause of silent telemetry:** `shell: bash` in GitHub Actions runs as `bash -eo pipefail`. Both action files had:

```bash
# semver/action.yml line 35
LAST_TAG=$(git tag -l '...' | grep -E '...' | head -1)

# release-notes/action.yml line 34
PREV_TAG=$(git tag -l '...' | grep -E '...' | grep -v ... | head -1)
```

When the repo has no matching semver tags (the normal state), `grep` exits 1 (no matches), pipefail propagates, and the script exits before doing any work. This is why every `main` push fails and production has never been updated, which is why PostHog telemetry has been silent 35+ days.

**Fix:** Added `|| true` to both pipelines. PR #151 from a prior session proposed the same fix but was open/unmerged. Incorporated into this session's branch and closed stale PRs #152–#157.
