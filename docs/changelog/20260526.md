## Reasoning

Write-only fields are a quiet correctness hazard: they imply the struct can be reconfigured or its behavior varies with the stored value, when in fact it never reads them. They also mislead future developers about what state is active at runtime. Removing them makes the contract honest and reduces the struct surface area.

The `insights` fields were discovered by noticing the same `cfg *config.Config` pattern in both `Reporter` and `Analyzer` — neither struct has any method that dereferences `r.cfg` or `a.cfg` after construction.

**Single incident:** at 02:57:12 UTC a `POST /api/v1/pull/teller` was called while 8+ Teller `transactions.processed` webhooks arrived simultaneously. The long-running pull held the file lock; webhooks timed out (each after ~2 minutes, from 02:59 to 03:06). All errors resolve once the lock is released.

**`errors=5` in sync:** Production log shows `errors=5` (up from `errors=4` pattern). The `ba2387f` fix is in `probably` HEAD but may not be in the `personal-cfo` binary. Cannot confirm from this repo.

### Reasoning

The `personal-cfo` codebase uses file-based locking for a "raw transaction store" (`/data/cfo/.locks/teller-transactions.lock`). The `probably` rewrite uses PostgreSQL — no file locks. The production errors are not directly reproducible in `probably`.

However, analogous weaknesses exist in `probably`:
1. **No timeout on Teller webhook goroutine** — `context.Background()` with no deadline; goroutine leaks if Teller API or DB stalls
2. **Dead `s.cfg != nil` guard on `QueueForEnrichment`** — `QueueForEnrichment` is a DB call that never reads config; the guard is always `true` in production but silently skips enrichment if cfg is somehow nil (silent data-loss bug)

### Shipped — PR #682 (merged to main at `be952d89`)

**`internal/handlers/teller.go`** — wrap the `transactions.processed` background goroutine with `context.WithTimeout(context.Background(), 2*time.Minute)`. Addresses `TellerWebhookTransactionAggregationError` pattern: hung goroutines under concurrent webhook load.

**`internal/sync/teller.go`** — remove `&& s.cfg != nil` guard from `QueueForEnrichment` calls in `SyncTransactions` (line 819) and `DeleteAndResync` (line 990). Addresses `ProviderTransactionStoreError` pattern: silent enrichment skip when cfg somehow nil.

**`internal/sync/plaid_sync.go`** — same guard removal in `PlaidSyncTransactions` (line 741).

Net: **−3 dead conditions**, +1 timeout boundary.
