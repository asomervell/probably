### Root cause: `bash -eo pipefail` + `grep` exits 1 on no-match

Both `.github/actions/semver/action.yml` and `.github/actions/release-notes/action.yml` run under
`shell: bash` (GitHub Actions default: `bash -eo pipefail`). When the repo has no semver tags,
`grep` exits 1, `pipefail` propagates that through the command substitution, and the script dies
before the downstream `[ -z "$LAST_TAG" ]` guard ever runs. **Every push to `main` has been
silently failing at the semver step**, which is why production has never deployed and PostHog
telemetry has been silent 35+ days.

**Fix:** Added `|| true` to the `LAST_TAG` and `PREV_TAG` pipeline expressions in both actions.

PR #163 is open and correct. Needs owner merge.
